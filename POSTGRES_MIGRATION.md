# SQLite -> PostgreSQL Migration

This project currently stores data in `bookings.db` (SQLite). Use the migration script below to copy all tables/data to PostgreSQL.

## 1) Install dependency

```bash
pip install -r requirements.txt
```

## 2) Set PostgreSQL URL

```bash
export DATABASE_URL='postgresql://USER:PASSWORD@HOST:5432/DBNAME'
```

## 3) Run migration

```bash
python scripts/migrate_sqlite_to_postgres.py --sqlite-path bookings.db --postgres-url "$DATABASE_URL"
```

If `DATABASE_URL` is set, you can omit `--postgres-url`:

```bash
python scripts/migrate_sqlite_to_postgres.py --sqlite-path bookings.db
```

## 4) Verify

Check row counts in PostgreSQL for key tables:
- `users`
- `bookings`
- `resume_reports`
- `coding_submissions`
- `user_events`

## Notes
- Script truncates target Postgres tables before copying (safe for one-time migration, not incremental sync).
- Column types are mapped conservatively (`INTEGER` -> `BIGINT`, `REAL` -> `DOUBLE PRECISION`, `TEXT` -> `TEXT`).
- Integer primary keys are created as `GENERATED BY DEFAULT AS IDENTITY` in PostgreSQL.
